## 摘要
将推理与 API 调用交替进行的工具集成智能体在处理复杂任务方面具有前景，但针对高风险、特定领域的部署进行对齐仍具挑战：现有的强化学习方法依赖于粗粒度的二元奖励，无法区分工具选择错误与参数格式错误。我们提出了 ToolRLA，这是一个针对特定领域工具智能体的三阶段后训练流水线（SFT -> GRPO -> DPO）。核心贡献是一个具有乘法正确性分解的细粒度奖励函数，涵盖四个维度——格式有效性、工具选择、参数准确性和合规性——该函数将领域优先级顺序作为归纳偏置编码到奖励景观中。在金融咨询助手（80 多名顾问，每日 1,200 多次查询）上部署三个月后，ToolRLA 实现了：任务完成率提升 47%（62%->91%），工具调用错误减少 63%（38%->14%），合规违规减少 93%（12%->0.8%），且延迟低于 2 秒。消融实验表明，乘法奖励设计相比加法替代方案提升了 7 个百分点。泛化性在 ToolBench 和 API-Bank 上得到了进一步验证。

## Abstract
Tool-integrated agents that interleave reasoning with API calls are promising for complex tasks, yet aligning them for high-stakes, domain-specific deployment remains challenging: existing reinforcement learning approaches rely on coarse binary rewards that cannot distinguish tool selection errors from malformed parameters. We present ToolRLA, a three-stage post-training pipeline (SFT -> GRPO -> DPO) for domain-specific tool agents. The core contribution is a fine-grained reward function with multiplicative correctness decomposition spanning four dimensions -- format validity, tool selection, parameter accuracy, and regulatory compliance -- that encodes domain priority orderings as inductive biases in the reward landscape. Deployed on a financial advisory copilot (80+ advisors, 1,200+ daily queries), ToolRLA achieves over three months: a 47% improvement in task completion rate (62%->91%), a 63% reduction in tool invocation errors (38%->14%), and a 93% reduction in regulatory violations (12%->0.8%), within sub-2-second latency. Ablation studies show the multiplicative reward design accounts for 7 percentage points of improvement over additive alternatives. Generalization is further validated on ToolBench and API-Bank.

*   **核心问题**：在金融等高风险、专业化领域部署工具集成智能体（Tool-integrated agents）时，现有的强化学习（RL）对齐方法存在局限。

    *   **级联错误**：传统的“意图识别->槽位填充->路由”流水线缺乏纠错能力，一步错步步错。
    *   **合规性挑战**：金融领域对监管极其敏感，模型必须在保证任务完成的同时，严格遵守合规边界（如不提供收益保证）。
*   **整体含义**：论文提出了一种名为 **ToolRLA** 的三阶段后训练框架，通过创新的**乘法奖励分解机制**，将领域知识和优先级（合规 > 正确 > 效率）编码进奖励函数，显著提升了智能体在复杂、受限环境下的表现。

### 2. 方法论：核心思想与关键技术
ToolRLA 采用 **SFT -> GRPO -> DPO** 的三阶段流水线：
*   **阶段 1：SFT 冷启动**：利用 4.2K 条经沙盒验证的轨迹（包含 LLM 蒸馏、专家标注和日志重写）建立基础的工具调用能力。
*   **阶段 2：基于 GRPO 的工具对齐（核心）**：
    *   **GRPO 算法**：使用组相对策略优化（Group Relative Policy Optimization），通过组内平均奖励估计优势函数，无需价值网络，节省算力。
    *   **乘法奖励分解**：总奖励 $R = R_{fmt} + R_{cor} + R_{eff} + R_{cpl}$。
        *   **正确性奖励 ($R_{cor}$)**：采用乘法逻辑 $S_{name} \times S_{comp} \times S_{acc}$。如果工具名称选错（$S_{name}=0$），则整个正确性分数为 0（一票否决制），防止模型通过刷参数准确度来补偿工具选择错误。
        *   **合规性奖励 ($R_{cpl}$)**：设置巨大的负惩罚（$\lambda=10$），确保任何违规行为的得分都低于非违规行为。
*   **阶段 3：DPO 合规对齐**：针对难以用规则定义的“灰色地带”（如暗示性建议），利用专家标注的偏好对进行直接偏好优化，进一步压制违规风险。

*   **实验规模**：

### 6. 主要结论与发现
*   **乘法奖励的优越性**：乘法分解比加法分解在工具调用错误率（TIER）上提升了 **7 个百分点**，有效解决了模型“投机取巧”的问题。
*   **三阶段的必要性**：SFT 提供基础，GRPO 提升工具使用质量，DPO 解决复杂的合规边界，缺一不可。
*   **生产效益显著**：任务完成率（TCR）从 62% 提升至 91%，违规率从 12% 降至 0.8%，顾问满意度从 3.1 升至 4.3/5。
*   **模型选择**：14B 模型通过 ToolRLA 训练后，能缩小与 32B/70B 模型的性能差距，且满足实时性要求。

### 7. 优点
*   **逻辑严密**：乘法奖励设计非常符合工具调用的本质逻辑（选错工具，参数再对也无意义）。
*   **工业价值高**：针对高风险领域的合规性问题提供了系统性的对齐方案，且有真实的线上部署数据支撑。
*   **训练效率**：利用 GRPO 避免了价值网络的开销，且细粒度奖励加速了收敛。

### 8. 不足与局限
*   **沙盒依赖**：奖励计算高度依赖于后端 API 沙盒的保真度和同步频率，若 API 定义变更而沙盒未更新，会导致奖励信号失效。
*   **标注成本**：DPO 阶段仍需要大量合规专家的手工标注，且存在 16% 的标注者不一致性。
*   **数据隐私**：核心测试集 FA-Bench 为内部数据，外部研究者无法完全复现其在金融场景的特定表现。
*   **模态限制**：目前仅支持文本输入，无法处理包含图表、扫描件等金融文档的多模态场景。
